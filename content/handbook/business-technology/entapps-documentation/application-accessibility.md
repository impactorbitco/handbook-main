### Overview

The GitLab CorpSec and Enterprise Applications teams are committed to ensuring accessibility across our applications for the widest possible audience. We actively work to support all team members by enabling services and features already available within the products we use.

### MacOS Features

CorpSec and Security Engineering has all of MacOS's accessibility features enabled by default with the exception of Apple Intelligence (pending review). Here is [Apple's guide to features enabled on the Mac](https://support.apple.com/guide/mac-help/get-started-mh35884/mac).

MacOS has features that support the following:

1. [Hearing](https://support.apple.com/guide/mac-help/hearing-mchlb4f015b1/15.0/mac/15.0) - Closed captions, Screen Flast and support for assistive hearing devices

2. [Mobility](https://support.apple.com/guide/mac-help/mobility-mchl654a7cc7/15.0/mac/15.0) - Voice Commands and shortcuts, eye tracking and use of assistive input devices and switch controls.

3. [Speech](https://support.apple.com/guide/mac-help/speech-mchl6972f04a/15.0/mac/15.0) - Live Speech, Personal Voice

To enable these features, go to System Settings and select Accessibility. For any assistance, contact #it_help on Slack.

### WCAG Compliant Applications

Most SaaS applications support the Web Content Accessibility Guidelines (WCAG) international standard as part of their application build. This allows them to seamlessly interact with MacOS and Browser capabilities, assistive devices and peripherals in make their application much accessible. The following applications are WCAG Compliant:

1. Salesforce
2. ServiceNow
3. Coupa
4. Netsuite
5. UIPath
6. Workato
7. Tropic
8. Supplier.IO

## PSIRT Runbooks

This section contains operational runbooks for the Product Security Incident Response Team (PSIRT).

### Critical Incident Response

- [Handling priority::1/severity::1 Issues](/handbook/security/product-security/psirt/runbooks/handling-s1p1/) - Process for handling critical security incidents
- [Working with SIRT](/handbook/security/product-security/psirt/runbooks/working-with-sirt/) - Guide for engaging and collaborating with SIRT

### General Runbooks

- [CVSS Calculation](/handbook/security/product-security/psirt/runbooks/cvss-calculation/) - Guide for calculating CVSS scores on GitLab security issues
- [General process for the application security team in patch releases](/handbook/security/product-security/psirt/runbooks/security-engineer/) - Process for AppSec team involvement in security patch releases
- [Handling unintended vulnerability disclosures](/handbook/security/product-security/psirt/runbooks/unintended-vuln-disclosure/) - Runbook for handling different scenarios of unintended vulnerability disclosures
- [HackerOne Process](/handbook/security/product-security/psirt/runbooks/hackerone-process/) - Process for managing vulnerability reports through GitLab's bug bounty program
- [Holiday and Friends and Family Day Coverage](/handbook/security/product-security/psirt/runbooks/holiday-coverage/) - Process for PSIRT coverage during holidays and Friends and Family days
- [How to handle upstream security patches](/handbook/security/product-security/psirt/runbooks/upstream-security-patches/) - Process for handling security patches in upstream dependencies
- [PSIRT Case Lifecycle](/handbook/security/product-security/psirt/runbooks/psirt-case-lifecycle/) - Description of how PSIRT manages vulnerability cases through their lifecycle
- [Verifying Security Fixes](/handbook/security/product-security/psirt/runbooks/verifying-security-fixes/) - Process for validating and verifying security fixes

## Note for Team Members

When you're on-call or handling a security incident and need help or advice, reach out in the `#security-department` Slack channel or escalate according to the [Security Engineer On-Call process](/handbook/security/security-operations/sirt/engaging-security-on-call/).

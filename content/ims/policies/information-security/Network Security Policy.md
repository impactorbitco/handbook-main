# Network Security Policy

## 1. Purpose

The purpose of this Network Security Policy is to establish a strategic direction and framework for maintaining the confidentiality, integrity, and availability of the company's network and associated information assets. This policy aligns with the ISO/IEC 27001 standards for an Information Security Management System (ISMS) and supports compliance with organizational and regulatory requirements.

## 2. Scope

This policy applies to all employees, contractors, and third-party users who have access to the organization's network infrastructure. It encompasses all hardware, software, and data assets within the organization, as well as external connections and cloud services.

## 3. Objectives

- Ensure the security of the network infrastructure and protect it from unauthorized access, misuse, and attacks.
- Safeguard sensitive information stored, transmitted, or processed within the network.
- Promote a culture of security awareness among employees and stakeholders.
- Comply with ISO/IEC 27001 standards and other relevant regulatory frameworks.

## 4. Roles and Responsibilities

### 4.1. Owner

The owner of this policy is Ryan Laird, who is responsible for its implementation, maintenance, and periodic review.

### 4.2. Employees and Users

All employees, contractors, and third-party users must comply with this policy and report any suspected security incidents to the designated security team.

### 4.3. Security Team

The security team is responsible for monitoring network activity, responding to incidents, and maintaining the security of the network infrastructure in alignment with this policy.

## 5. Key Requirements

### 5.1. Access Control

Access to the network must be restricted to authorized users only. Multi-factor authentication (MFA) should be implemented for critical systems to enhance security.

### 5.2. Data Protection

All sensitive data transmitted over the network must be encrypted using industry-standard protocols, such as TLS or IPSec. Regular data backups must be conducted and securely stored.

### 5.3. Network Monitoring

The organization must implement continuous monitoring of network traffic to identify and mitigate potential threats. Logs must be maintained for a minimum duration as specified by regulatory requirements.

### 5.4. Incident Response

An incident response plan must be in place to address any security breaches or vulnerabilities. This plan must include procedures for containment, eradication, recovery, and post-incident analysis.

### 5.5. Physical Security

Network equipment, such as servers and routers, must be physically secured in compliance with Annex A8 Control Group standards.

## 6. Policy Review and Maintenance

This policy must be reviewed on an annual basis or whenever significant changes occur in the network infrastructure or regulatory requirements. The last review was conducted on May 1, 2025, by Ryan Laird.
